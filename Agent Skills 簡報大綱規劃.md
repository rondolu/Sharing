## **Slide 3: 痛點回顧 —— 為什麼模型很聰明，但還是常常出錯？**

* **摘要**：  
  * **現況**：就算我們用最好的大語言模型，每次要 AI 幫忙檢查 SQL 語法或 Python 程式碼風格時，輸出的品質常常起伏不定，很看當下 Prompt 寫得好不好。  
  * **原因**：現在的 AI 就像一個「IQ 300 但沒看過我們公司 codebase 的數學天才」，如果你不給他公司內部的標準規範，他就會用猜的，進而產生幻覺或給出不合規的建議。  
  * **解法**：我們需要的不是更聰明的天才，而是一位嚴格遵守 SOP 的「資深審核員」，這就是 Agent Skills 要解決的問題。

## **Slide 4: 概念對齊 —— 用資料工程的角度看 AI 架構**

* **摘要**：  
  把 AI 的運作機制想像成我們熟悉的資料管線（Data Pipeline）：  
  * **Models（處理器）**：就像**運算引擎（如 Spark 叢集）**。算力極強，但本身不知道你要處理什麼商業邏輯。  
  * **Agents（代理作業系統）**：就像**排程器（如 Airflow）**。負責管理記憶體、決定什麼時候該觸發任務、協調各項資源。  
  * **MCP（工具連接器）**：就像 **Database Connectors 或 API SDK**。提供物理管道讓 AI 可以連線去撈外部的資料庫或檔案。  
  * **Skills（技能包）**：就是我們寫好的 **ETL 腳本與 SOP 設定檔**。它是一包資料夾，明確定義了「先撈什麼資料、接著套用什麼 Python 檢查規則、最後產出什麼格式的報告」，引導 AI 穩定完成任務。

## **Slide 5: Skills 怎麼運作？—— 不用每次都重貼 Prompt**

* **摘要**：  
  * 以前寫 Prompt，每次都要把幾千字的規則貼進對話框，AI 容易忘記而且很浪費 Token。  
  * Skills 是一個標準化的「資料夾」，採用「按需載入（需要用到才讀取）」的機制。  
  * **三層架構**：  
    1. **身分證 (YAML)**：永遠常駐在背景，只有短短幾行，告訴 AI 這個技能是做什麼的。  
    2. **SOP 本體 (SKILL.md)**：當 AI 發現使用者現在的需求跟這個技能吻合，才會把完整的工作步驟讀進來。  
    3. **輔助檔案 (scripts/ 或 references/)**：走到特定步驟時，才去跑裡面的 Python 腳本或查閱長篇的參考文件。

## **Slide 6: 把模糊的指令，變成有畫面的「檢查清單」**

* **摘要**：  
  寫 SKILL.md 最重要的一步，就是把人話轉成具體的步驟（SOP）。  
  * ❌ **錯誤寫法（太模糊）**：「分析這份資料」或「幫我做 Code Review」。  
  * ✅ **正確寫法（明確的驗證關卡）**：「1. 檢查表格的『Total\_Revenue』欄位有沒有空值。2. 計算月成長率。如果成長率低於 5%，必須觸發警報規則，並在報告最上方加上紅字警告」。  
  * 我們甚至可以把公司常用的 Python 或 SQL 檢查腳本直接丟進 scripts/ 資料夾，讓 AI 直接執行這些腳本，而不是讓 AI 自己隨機瞎編程式碼。

## **Slide 7: 實作 SOP：動手打造第一個技能包**

* **摘要**：  
  1. **建資料夾**：建立一個名稱為英文小寫加橫線的資料夾（例如 code-review-helper）。  
  2. **寫身分證**：在裡面建立一個檔案，檔名必須是全大寫的 SKILL.md，並在最上方用 \--- 包住 YAML 格式的介紹。  
  3. **寫步驟**：在 YAML 下方，寫上這項技能的「目的、輸入格式限制、執行步驟 1-2-3、以及最後輸出的報表長怎樣」。  
  4. **打包上傳**：把整個資料夾壓成 .zip 檔，上傳到系統設定的 Skills 裡面。

## **Slide 8: 開發地雷：這些格式錯一個都不會動**

* **摘要**：  
  請把這頁當作上線前的 Checklist：  
  * **禁忌字眼**：技能名稱絕對不能包含 claude 或 anthropic 這些系統保留字。  
  * **資安防禦**：YAML 區塊內**絕對不准**寫入 \< 或 \> 這類 XML 標籤，會引發資安風險跟讀取錯誤。  
  * **檔名限制**：核心檔案只能叫 SKILL.md（不能小寫），而且資料夾裡面**不要**放 README.md，系統會被搞混。

## **Slide 9: 觸發控制：怎麼讓 AI 知道何時該出手？**

* **摘要**：  
  * 技能會不會被喚醒，全看 YAML 裡面的 description 怎麼寫。  
  * **要寫明觸發條件**：例如「當使用者上傳廣告報表 (CSV)，且要求計算 ROAS 時使用」。  
  * **加入負向約束（不要做什麼）**：這點很重要！要在描述裡加上防呆機制，例如「如果使用者傳的不是 CSV 檔，就停止執行」，這樣才不會浪費 Token 或讓 AI 亂回答。

## **Slide 10: 結語**

* **摘要**：  
  * 將日常繁瑣的工作（如固定的資料清洗步驟或程式碼風格審查）寫成 Skills，能確保每次執行的品質都一樣標準。  
  * 這是一勞永逸的投資：只要團隊裡有一個人把某個技能寫好並上傳，全團隊的 AI 助手就能直接使用這個標準流程。  
  * **Action Item**：請大家盤點手邊「最常重複下相同 Prompt」的工作，本週試著把它寫成第一個 SKILL.md 資料夾。
